Toast notifications are transient messages used to provide toast for user-initiated actions. A toast notification:

  • should appear at the top center of product consoles or above the referred UI location.
  • is often triggered by user actions.
  • should be closable.
  • can automatically close itself after being displayed for 5 seconds.
  • should never be overlapped by other toast notifications. An older notification needs to be closed before the next one is displayed.

Types:

  • Error: This type of toast notification is used to indicate incorrect or unsuccessful user actions.
  • Warning: This type of toast notification is used to indicate unusual user actions.
  • Success: This type of toast notification is used to indicate successful user actions.
  • Informational: This type of toast notification is used to provide additional information on user-initiated actions.

For detailed information, see Toast Messages.
